# Requisitos Funcionais

**Descrição:**  
Os requisitos funcionais definem as ações, os comportamentos e as funcionalidades específicas que a plataforma **Radar UnB** deve possuir para que os usuários consigam realizar suas tarefas e alcançar seus objetivos dentro do sistema.

---

## Lista de Requisitos Funcionais (RF)

* **RF01** – O sistema deve permitir o cadastro de usuários.
* **RF02** – O sistema deve permitir login e autenticação de usuários.
* **RF03** – O sistema deve exibir um mapa interativo do campus da UnB.
* **RF04** – O sistema deve permitir ao usuário registrar problemas com localização geográfica.
* **RF05** – O sistema deve permitir ao usuário cadastrar eventos no mapa.
* **RF06** – O sistema deve permitir anexar imagens e descrições às ocorrências.
* **RF07** – O sistema deve permitir filtrar ocorrências por tipo (problema ou evento).
* **RF08** – O sistema deve permitir filtrar ocorrências por localização e data.
* **RF09** – O sistema deve exibir detalhes ao clicar em um ponto no mapa.
* **RF10** – O sistema deve permitir que usuários validem ou confirmem ocorrências.
* **RF11** – O sistema deve permitir acompanhar o status de um problema (ex: pendente, resolvido).
* **RF12** – O sistema deve permitir editar ou excluir registros feitos pelo próprio usuário.
